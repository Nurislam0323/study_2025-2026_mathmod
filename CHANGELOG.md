## Лабораторная работа №5 — «Математическое моделирование: модель хищник–жертва»

**Автор:** Закиров Нурислам (1132236040@rudn.ru)
**Организация:** Российский университет дружбы народов
**Лицензия:** CC BY

### Новое

- **Модель Лотки–Вольтерры** — реализована классическая система «хищник–жертва»:
  - Базовая модель: `dx/dt = -ax + bxy`, `dy/dt = cy - dxy`
  - Расширенная модель с внутривидовым ограничением: `dx/dt = -ax + bxy - kx²`
  - Стационарное состояние: `(x₀, y₀) = (a/b, c/d) = (10, 10)`

- **Численное решение ОДУ** — использован пакет `DifferentialEquations.jl` (алгоритм `Tsit5`):
  - Временные графики `x(t)`, `y(t)` на интервале `[0, 150]`
  - Фазовые портреты `x(y)` для обеих моделей
  - Начальные условия: `x₀ = 8`, `y₀ = 11`

- **Параметрическое сканирование** — исследование влияния параметров:
  - Базовая модель: варьирование `a ∈ {0.15, 0.25, 0.35, 0.50}`
  - Расширенная модель: варьирование `k ∈ {0.0, 0.001, 0.002, 0.005}`
  - Сравнительные графики траекторий и фазовых портретов

- **Аналитические метрики**:
  - `norm_final = √(x(t_final)² + y(t_final)²)` — характеристика конечного состояния
  - Бенчмаркинг времени вычислений для всех комбинаций параметров

- **Визуализация результатов** — 9 графиков:
  - Временные зависимости для базовой и расширенной моделей
  - Фазовые портреты (замкнутые кривые vs. сходящиеся спирали)
  - Сравнительные графики параметрического сканирования
  - Зависимость `norm_final` и времени вычислений от параметров

- **Воспроизводимая структура проекта** — использован фреймворк `DrWatson.jl`:
  - Модульные QMD-файлы (`01_lotki.qmd`, `02_lotki.qmd`)
  - Автоматическое сохранение данных (`CSV`, `JLD2`) и графиков (`PNG`)
  - Параметрическая функция `produce_or_load` для кеширования результатов

### Основные выводы

1. Базовая модель демонстрирует устойчивые незатухающие колебания (замкнутые фазовые траектории)
2. Расширенная модель стабилизируется к равновесию (спиральные фазовые траектории)
3. Параметры `a` и `k` влияют на форму, частоту и скорость затухания колебаний
4. Вычислительные затраты остаются малыми для обеих моделей

### Технологии
